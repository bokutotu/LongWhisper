# Commit message template

Use this structure when the repository prefers full commit messages:

```text
<type>(<scope>): <summary>

Why:
- <motivation>

What:
- <change summary>

Validation:
- <command or verification detail>

Notes:
- <caveat or follow-up>
```

Guidance:

- Omit any section that would be empty or add only filler.
- Do not leave placeholder bullets in the final message.
- Keep the subject line specific and scoped to the actual change.
- Prefer exact commands for `Validation:` when available.
- Use `Notes:` only for caveats, skips, or environment limits worth preserving.
